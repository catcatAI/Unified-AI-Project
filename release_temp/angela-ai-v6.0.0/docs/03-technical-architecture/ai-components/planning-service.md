# 規劃服务 (Planning Service)

## 概述
規劃服务 (`planning_service.py`) 負責根據目標和約束生成和優化計畫。它提供了一個統一的介面，用於執行規劃請求，並抽象化了底層規劃演算法或模型的細節。

## 當前狀態 (模擬)
目前，規劃服务處於**模擬狀態**。它不與任何真實的規劃演算法或 LLM 進行互動，而是根據輸入目標、約束和上下文返回預設的模擬計畫。這使得 Agent 可以在不依賴外部規劃服務的情況下進行開發和測試。

## 核心組件
- **`PlanningManager`**: 負責管理規劃請求。它提供了一個 `generate_plan` 方法，用於生成計畫並返回結果。

## 整合點
- **`planning_service.py`**: 包含 `PlanningManager` 的核心邏輯。

## 未來發展
- **真實演算法整合**: 將 `PlanningManager` 擴展為整合真實的規劃演算法 (例如 STRIPS, PDDL 求解器) 或專門的規劃 LLM。
- **複雜約束處理**: 支援更複雜的約束類型和優化目標。
- **動態重新規劃**: 實現根據環境變化進行動態重新規劃的能力。
