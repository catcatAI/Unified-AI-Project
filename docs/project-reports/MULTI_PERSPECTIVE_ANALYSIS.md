# 🧠 Angela AI 多維度架構分析

本文件從使用者、開發者與系統安全三個維度，深度解析 Angela AI 的設計哲學與技術實現。

## 1. 使用者維度 (User Perspective)
*「Angela 不僅是工具，更是伴侶。」*

- **無感安全**: 使用者只需啟動監控器，系統自動處理複雜的密鑰生成與同步，無需手動配置加密參數。
- **跨端體驗**: 無論在電腦前還是外出使用手機，Angela 的記憶與情感狀態始終保持同步。
- **直觀交互**: 透過 Live2D 視覺反饋與語音交互，降低了人機溝通的隔閡感。

## 2. 開發者維度 (Developer Perspective)
*「模組化、可擴展、高效能。」*

- **狀態矩陣 (V×L×P×M)**: 統一的數據結構讓各個子系統（視覺、聽覺、邏輯）能共享相同的狀態語境。
- **分層架構**: 
    - `L1-L2`: 基礎感官與短期記憶。
    - `L3-L4`: 身份識別與長期認知。
    - `L5-L6`: 複雜推理與多模態決策。
- **硬體適應**: 系統會自動探測 CPU/GPU 能力，動態調整 Live2D 幀率與 AI 模型精度。

## 3. 安全維度 (Security Perspective)
*「物理隔離、邏輯校驗。」*

- **A/B/C 密鑰隔離**:
    - **Key A**: 控制面安全。
    - **Key B**: 通訊面安全。
    - **Key C**: 數據面安全。
- **HMAC 簽名校驗**: 行動端請求必須攜帶基於 Key B 的 HMAC-SHA256 簽名，杜絕非法調用。
- **常駐監控**: 透過 `SecurityTrayMonitor` 實現對後端進程的實時監控與重啟保護。

1. **更深層的矩陣成熟度 (L7-L11)**: 實現真正的自我演化與代碼自修復能力。
2. **完全去中心化的集群**: 讓多台閒置裝置共同支撐一個更強大的 Angela 實體。
3. **生物訊號整合**: 透過穿戴裝置獲取使用者生理數據，實現更精準的情感共鳴。
