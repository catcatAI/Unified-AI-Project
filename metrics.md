# Angela AI v6.2.0 - Performance Metrics (性能指标)

本文件记录 Angela AI v6.2.0 在不同硬件环境下的性能指标、推算表现及实测数据。

## 1. 统一状态矩阵 (Unified Status Matrix: V × L × P × M)

Angela 的运行状态由四维张量共同定义，实现了从「版本能力」到「运算模式」的全面整合。

| 维度 | 代号 | 范围 (Range) | 说明 |
| :--- | :--- | :--- | :--- |
| **Version Status** | **V** | v6.2.0 - Production | Angela 目前的版本功能状态与能力边界 |
| **Maturity Level** | **L** | L0 ~ L11 | 12 个阶段的矩阵成熟度，定义认知深度 |
| **Precision Level** | **P** | FP8 ~ FP128 | 5 个等级的浮点精度，定义数值解析度 |
| **Precision Mode** | **M** | INT ~ DEC4 | 5 个模式的精度模式，定义整数/小数传输策略 |

**状态公式**: `Angela_State = V ⊗ L ⊗ P ⊗ M`

## 2. 安全与通讯监控 (Security & Communication Matrix)

为了确保跨装置（桌面/行动）通讯的安全，Angela 引入了 A/B/C 密钥体系与常驻监控器，并与外部模型密钥进行了物理与逻辑上的隔离。

### **2.1 密钥 vs 密钥 (Secret Keys vs API Keys)**

| 类别 | 对象 | 存储位置 | 生命周期 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **系统密钥 (Angela Secrets)** | Key A, B, C | `data/security/abc_keys.json` | 系统生成，可重置 | 用于内部通讯、行动端验证、后端控制 |
| **外部密钥 (Model API Keys)** | OpenAI, Anthropic, etc. | `.env` 或 `configs/*.yaml` | 外部服务商提供 | 用于调用大模型 API，与系统安全性解耦 |

### **2.2 A/B/C 密钥体系详解**

| 密钥类型 | 代号 | 用途 | 保护范围 |
| :--- | :--- | :--- | :--- |
| **Key A** | **Backend Control** | 后端服务启停与核心权限 | 本地系统管理 (System Tray) |
| **Key B** | **Mobile Comm** | 行动端与后端加密通讯 (HMAC-SHA256) | /api/v1/mobile/* |
| **Key C** | **Sync/Desktop** | 桌面端同步与跨装置二级验证 | 全域同步数据 |

**监控表现 (推算)**:
- **密钥隔离效率**: 100% (独立管理路径)
- **密钥生成速度**: < 50ms (Fernet/AES-128)
- **通讯验证延迟**: < 2ms (HMAC-SHA256)

## 4. 精度图谱 (Precision Map)

针对不同功能模块，系统采用差异化维度与精度策略，以平衡性能与效能。

| 模块 (Module) | 预设维度 (Shape) | 预设精度 (Precision) | 动态范围 (Range) | FP 切换支持 |
| :--- | :--- | :--- | :--- | :--- |
| **Vision (视觉)** | (8, 8) | FP16 | FP8 - FP32 | 否 |
| **Audio (音频)** | (4, 4) | FP8 | FP8 - FP16 | 否 |
| **Logic (逻辑)** | (8, 8, 8, 8) | FP32 | FP16 - FP64 | 是 (FP128->FP8) |
| **Memory (记忆)** | (12, 8) | FP64 | FP32 - FP128 | 是 (FP128->FP8) |
| **Sensory (感官)** | (4, 4) | FP8 | FP8 - FP16 | 否 |
