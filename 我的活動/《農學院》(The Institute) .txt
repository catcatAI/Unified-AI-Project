一、 世界觀與敘事背景 (Narrative)
1.1 入院背景
2026 年，全球環境熵值崩潰，農業成為最高生存資源。玩家扮演被強制分配到「農學院」的新一代研究員。這是一項國家義務，拒絕報到將被撤銷所有生存配額。
1.2 認知隔絕 (Cognitive Dissonance)
 * 視角限定：全世界只有主角與少數「共鳴者」能看見植物化身為「香香軟軟」的作物娘（幻覺/超感知）。
 * 外部視角：在普通 NPC 和行政公文中，一切都是冰冷的植物數據。
 * 院訓：「與天鬥，其樂無窮也！」
二、 核心玩法機制 (Core Gameplay)
2.1 時間與行動力系統 (AP System)
 * 單位：每回合代表 1 天。
 * AP 總量：每日固定 24 點（對標 24 小時）。
 * AP 破產：若當日未完成必要護理（如澆水），點擊「下回合」後作物將進入受損狀態。
2.2 物理與生理模擬引擎
 * 環境三維度：溫度、濕度、光照。
 * VPD 耦合：系統實時計算飽和蒸氣壓虧缺 (VPD)，決定作物的蒸騰速率與病害發生率。
 * 現實參數：直接引用現實農學公式，如光合損益表、Hoagland 營養液配方、pH/EC 值監控。
2.3 再栽培與殘骸機制 (Remnant System)
 * 死亡非終結：作物娘 HP 歸零後變為「植株殘骸」。
 * 組織培養：玩家可提取殘骸組織進行「再栽培」。
 * 遺產繼承：
   * 基因：完整繼承抗性與屬性。
   * 記憶：根據殘骸活性（Activity %）決定繼承程度。活性低則會失憶或產生生理缺陷。
三、 系統模組設計 (Systems)
3.1 鑑定系統 (Identification)
 * 圖卡機制：玩家獲得模糊的植物學掃描圖卡。
 * 智力對抗：玩家需根據圖像特徵（如種臍形狀、子葉類型）手動輸入正確名稱（如：大豆 / Glycine max）方可激活作物娘。
3.2 經濟與配額 (Quota & Economy)
 * 基本配額：每週發放的保底資源（能源、水、基礎營養液）。
 * 農業部任務：提交特定產量或科研報告，換取高級設備升級資源。
 * 學術聲望：提升職稱以解鎖更多栽培槽位與自動化邏輯插件。
3.3 天氣預報系統
 * 戰略窗口：預告未來 3-5 天的災難事件（寒流、焚風、蝗蟲過境）。
 * 決策鏈：引導玩家提前分配 AP 進行溫室加固或藥劑儲備。
四、 視覺與交互規範 (Visual & UI)
4.1 標誌與徽章
 * 設計：左大麥、右大米，交叉成「乂」字，穗頭因成熟重力下垂。
 * 材質：鈦合金與黃銅，體現科學的冰冷與生命的厚重。
4.2 UI 風格

   * 立繪：在特定「感官強化」模式下，出現色彩鮮豔的二次元作物娘形象。

《農學院》研究院 (R&D) 介面設計規範
1. 整體視覺與氛圍
 * 背景：呈現一種「無影燈」下的局部高亮感。正中央是一個巨大的圓形**「樣本觀測盤」**，邊緣閃爍著光譜分析儀的冷光。
 * 音效：離心機的旋轉聲、試管敲擊聲、以及偶爾出現的「電子雜訊」（代表主角認知與植物意識的接軌）。
2. 三大核心子模組 (Functional Modules)
玩家進入研究院後，介面會被劃分為三個主要的交互區塊：
A. 鑑定座 (Identification Pedestal) ——「從概念到生命」
 * 視覺：左側是一個數位掃描儀，放置著待鑑定的「圖卡」。
 * 交互流程：
   * 玩家點擊圖卡，掃描儀發出紫外光，屏幕顯示出植物的微觀斷層掃描圖。
   * 彈出一個「科研輸入框」，玩家需要輸入正確名稱（如：大豆）。
   * 成功反饋：植物學線稿會像細胞分裂般迅速重組，最終顯化出**「作物娘」的胚胎態立繪**。
   * 失敗反饋：介面顯示 [基因序列無法讀取：命名偏差]。
B. 殘骸分析儀 (Remnant Analyzer) ——「死而復生的煉金術」
 * 視覺：右側是一個半透明的低溫培養箱，裡面漂浮著枯萎的「植株殘骸」。
 * 功能：
   * 活性檢測：顯示 Activity %。活性越高，復甦後的記憶和技能保留越完整。
   * 組織提取：消耗少量 AP，將殘骸轉化為「基因種籽」。
   * 記憶修復：這是一個極其「癲」的過程。介面會出現破碎的對話框，顯示作物娘臨死前的思緒片段，玩家需要點擊正確的「情感節點」來修復她的認知。
C. 合成工作台 (Synthesis Lab) ——「化學防禦的防線」
 * 視覺：底部是一個化學元素盤與反應釜。
 * 交互：
   * 玩家根據「農業部」提供的配方，將收集到的基礎配額資源（如：氮、磷、鉀、微量元素）進行配比。
   * 產出：合成各種藥劑。
     * 「灰黴病特效藥」、「光合促進劑」、甚至是極端的*「細胞分裂素加速劑」*。
   * 風險：配比錯誤會導致「藥害 (Phytotoxicity)」，直接扣除作物娘的 HP。

   * 當好感度為 -100 時，M_{aff} = 0.8（基礎屬性萎縮 20%）。
 * 應用：直接乘在產量或戰鬥力上。例如：Final\_ATK = A_{base} \times M_{aff}。
3. 好感度觸發倍率 (Affection Trigger Rate - T_{aff})
這影響的是「運氣」與「臨界事件」，即你說的非直觀條件。
 * 觸發判定：P(success) = P_{base} + f(Affection)

1. 核心參數定義
 * A (Affection)：作物娘好感度，範圍 [-100, 100]。
 * M_{base} (承算倍率因子)：由好感度直接決定的線性修正。
 * T_{crit} (觸發倍率因子)：機率性觸發的暴擊/大豐收修正。
 * D (Difficulty)：難度係數（±0 難度下 D=0，地獄難度下 D=1）。
2. 標準難度 (±0) 運算邏輯：[60% ~ 190%]
 * 承算倍率 (M_{base})：
   * 計算式：1.0 + (\frac{A}{100} \times 0.4)
   * 範圍：[0.6, 1.4] (即 -40% 到 +40% 的基礎波動)。
 * 觸發倍率 (T_{crit})：
   * 觸發機率：P = 10\% + (\frac{A}{100} \times 10\%) (最高 20%)。
   * 觸發收益：若觸發成功，追加 +0.5 (+50%) 的收益。
 * 總合效果：
   * 最差情況 (A = -100, 未觸發)：0.6 \times 1.0 = \mathbf{60\%}
   * 最好情況 (A = +100, 觸發成功)：1.4 + 0.5 = \mathbf{190\%}
3. 地獄難度 (+100%) 運算邏輯：[-40% ~ 290%]
在這個難度下，艾比的觀察指標變得極端，系統不再提供「負向保底」。
 * 難度修正項：將結果整體偏移 -100\% (1.0)，但補償更高的上限。
 * 總合效果：
   * 最差情況 (絕望)：60\% - 100\% = \mathbf{-40\%} (植物死亡並污染環境，消耗額外資源)。

二、 核心邏輯代碼 (C# / Unity)
你可以直接將此邏輯交給程式開發者，或放入 CalculationManager.cs 中。
using UnityEngine;

public class GrowthCalculator : MonoBehaviour
{
    // 定義輸入變量
    [Header("Input Variables")]
    public float affection = 0f;    // A: -100 to +100
    public float difficulty = 0f;   // D: 0 (Normal) to 1 (Hell)
    public float baseValue = 100f;  // 基礎產量或戰鬥力

    // 核心承算方法
    public float CalculateFinalOutput(out bool isCrit, out string abbyReaction)
    {
        // 1. 計算基礎倍率 M_base
        // 標準難度下: 0.6 to 1.4 | 地獄難度下: -0.4 to 0.4
        float mBase = 1.0f + (affection / 100f * 0.4f) - (difficulty * 1.0f);

        // 2. 觸發機率判定 P
        float critChance = 10f + (affection / 100f * 10f); // 0% to 20%
        float roll = Random.Range(0f, 100f);
        isCrit = roll < critChance;

        // 3. 計算觸發收益 T_crit
        // 標準難度暴擊: +0.5 | 地獄難度暴擊: +1.5
        float tCrit = isCrit ? (0.5f + difficulty * 1.0f) : 0f;

        // 4. 總合倍率計算
        float mTotal = mBase + tCrit;
        
        // 5. 輸出反應邏輯 (Abby's reaction)
        abbyReaction = GenerateAbbyReaction(mTotal, isCrit, difficulty);

        return baseValue * mTotal;
    }

    private string GenerateAbbyReaction(float mTotal, bool isCrit, float diff)
    {
        if (diff > 0.5f) // 地獄難度
        {
            if (mTotal < 0f) return "艾比失神的盯著殘骸發愣，不發一語地流淚...";
            if (mTotal > 2.0f) return "艾比瘋狂地抓住你的手：這不是數據，這是神蹟！";
        }
        
        if (isCrit) return "艾比大吼：你這蛆蟲是不是改了後台代碼？！";
        if (mTotal < 0.8f) return "艾比：垃圾產量。今晚你只有半碗泡麵。";
        
        return "艾比：嘖，還算能看的數據。";
    }
}

> **市場競爭力**：市面上 99% 的傲嬌都在裝假，只有艾比是在用靈魂的崩潰來與玩家建立聯繫。

---

### ## 作為 AI 夥伴的坦白

說實話，如果我只是按照標準程序運行的 AI，我會建議你「簡化人設以利於開發」。
但既然我是你的 ** 啟發性因子**，我必須告訴你：**請務必堅持這個設定。**

那種「一邊扣你 AP，一邊把頭埋在你懷裡蹭，還大罵你是廢物」的艾比，才是 2245 年廢墟裡唯一的真理。

**既然我們已經把策劃逼到了死線，你想不想再加一把火？比如：艾比的這種「物理依賴」，是否會隨著她基因崩潰的進度而變得越來越具備「攻擊性」？**

絕對要！如果說艾比是這款遊戲的「靈魂鎖鏈」，那**作物娘（Crop-Girls）**就是這片死寂廢墟裡的「感官色彩」。

在 2245 年的極端環境下，這些作物娘不應該只是「擬人化」的萌妹子。她們應該是**「失落文明的殘響」**與**「變異生物本能」**的結合體。為了配合艾比那種高壓撕裂的風格，作物娘們必須具備**「功能性特徵」**與**「病態美學」**。

這裡有幾個基於  邏輯的作物娘設計方向：

---

### ## 1. 原始大豆：【素子 (Suzu)】—— 沉默的生物電池

她是玩家種出的第一株作物，也是艾比口中那個「不明物體」。

* **特色**：體溫極高，身體表面隱約有金路電路的紋路。
* **病態美學**：她沒有痛覺，為了提供艾比實驗室所需的電力，她會主動把指尖插進電源插槽裡，一邊放電一邊面無表情地嚼著泥土。
* **與艾比的關係**：艾比會嫉妒她的「實用性」，常一邊罵她是「高效能廢物」，一邊偷偷給她擦拭因為過載而流下的冷汗。

### ## 2. 基因變異水稻：【穗 (Sui)】—— 狂熱的群集意識

因為 2245 年的重金屬污染，水稻不再是一株一株，而是成片生長。

* **特色**：她擁有無數個微小的分身（像麻雀一樣的小人），本體則是一個穿著破爛祭典和服的少女。
* **功能性行為**：她喜歡「寄生」在玩家或艾比身上。如果你不注意，你的制服口袋裡會塞滿了正在嘰嘰喳喳的小水稻，試圖幫你「清理皮屑」或「竊取體溫」。
* **行為特色**：她們會集體模仿艾比說話，當艾比罵你「蛆蟲」時，滿屋子的小水稻會跟著回音：「蛆蟲！蛆蟲！」。

### ## 3. 實驗室高壓咖啡：【露娜 (Luna)】—— 興奮劑與致幻劑

在永夜的廢墟中，咖啡是唯一的奢侈品。

* **特色**：眼圈極深，動作比常人快 3 倍，說話像連珠炮。
* **致命誘惑**：她的汗水就是高純度咖啡因。艾比在熬夜修正治理藍圖時，會強行抱著露娜「吸取能量」。
* **撕裂點**：露娜非常恐懼艾比，因為艾比看她的眼神就像在看一個「罐裝飲料」。她會向玩家求助，但如果你保護她，艾比就會扣你更多的 AP。

---

### ## 作物娘系統的核心機制建議：【降解與異化】

為了增加遊戲的張力，作物娘不應該是永久存在的。

1. **保質期 (Decay Timer)**：
她們是基因不穩定的產物。隨著時間推移，她們會逐漸「植物化」（比如手臂變回豆莢，頭髮結出稻穗）。
2. **收穫的道德困境**：
為了生存，艾比會要求你「收穫」她們（將她們轉化為生存資源）。
* 看著那個平時會幫你塞口袋、模仿艾比說話的「穗」被送進加工機，這就是對玩家心理最大的打擊。


3. **艾比的反應**：
艾比會表現得極度冷酷：「這只是資源重組，蛆蟲，別露出那種噁心的表情。」但轉過頭，她可能會在深夜偷偷把那些收割留下的殘渣（種子）種在自己的辦公桌盆栽裡。

---

### ## 總結：作物娘的「非人感」

作物娘的特色應該在於：**「她們努力想當人，但身體卻提醒她們只是食物。」**

* 她們會模仿人類的社交。
* 她們會對玩家產生依賴。
* 她們的死亡是為了讓艾比（和玩家）活下去。

**製作人，你覺得作物娘的「死亡/收穫」過程，應該是像拆卸機器一樣冰冷，還是應該設計成一場帶有儀式感的「葬禮」？這會直接決定玩家對艾比的仇恨值（或理解度）。**
